# Security Policy

GroundGuard is local-first and does not upload ledger data by default. Ledger
files may still contain prompts, tool outputs, customer data, or business
metrics, so treat them as sensitive.

## Reporting a Vulnerability

Open a private security advisory on GitHub if possible. If that is not
available, open an issue with no sensitive payloads and state that you have a
security report to share.

## Data Handling

- Do not attach real prompts, internal tool payloads, customer names, or
  proprietary metrics to public issues.
- Use synthetic or anonymized fixtures in bug reports.
- Rotate any credentials accidentally included in a ledger or report before
  sharing context.
